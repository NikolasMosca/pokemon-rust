# Game Loop Principale — 2026-05-22

## Flusso della run

Una run completa = 8 palestre. Ogni palestra segue questo ciclo:

```
[Battaglia] → vittoria → [Scelta: Pokécenter | Shop | Prossima battaglia]
                                                          ↑
                        (ripetere fino a 3 allenatori battuti)
                                                          ↓
                                               [Capopalestra]
                                                          ↓
                                          [Badge ottenuto] → prossima palestra
```

Game over → tutta la squadra KO.
Fine run → 8 badge ottenuti → messaggio "Sei pronto per la Lega Pokémon".

---

## Tipi di battaglia

| Tipo | Nemici | Cattura | Reward multiplier |
|---|---|---|---|
| Selvatico | 1 Pokémon random | ✅ sì | 1× |
| Allenatore | N Pokémon (= size squadra giocatore) | ❌ no | 2× |
| Capopalestra | N+1 Pokémon | ❌ no | 4× |

---

## Generazione nemici

### Pokémon selvatici
- Selezionati random da un pool filtrato per gym_index (livelli di area crescenti)
- Livello: media squadra giocatore ±2 (floor 5)

### Allenatori
- N Pokémon = `team.len()` del giocatore (min 1)
- Livello di ogni Pokémon: `media_squadra_giocatore ± 2` (random nel range)
- Specie: random dal pool dell'area
- Se il Pokémon a quel livello risulta evolved secondo la evolution_chain → usa forma evoluta
- Evolution chain caricata da PokéAPI e salvata in cache IDB (`api_cache`)

### Capopalestra
- Come allenatore ma `N = team.len() + 1`
- Livello: `media_squadra + 3` (fisso, più difficile)
- Reward multiplier: 4×

---

## Progressione palestra

```
gym_progress: {
    gym_index: 0..7,       // palestra corrente
    trainers_defeated: 0,  // reset a ogni nuova palestra
    pokecenter_uses: 0,    // reset dopo ogni capopalestra
    pokecenter_max: 3,     // massimo per palestra
}
```

Sequenza per palestra:
1. Battaglie selvatiche (illimitate, a scelta del giocatore)
2. Dopo ogni vittoria: scelta [Pokécenter | Shop | Prossima battaglia]
3. Dopo 3 allenatori battuti → il prossimo avversario è il capopalestra
4. Capopalestra battuto → badge +1, reset `trainers_defeated` e `pokecenter_uses`

---

## Pokécenter

- Heal completo di tutti i Pokémon del team (HP al massimo, nessun KO)
- Disponibile max `pokecenter_max` (3) volte per palestra
- Counter si resetta dopo aver battuto il capopalestra
- Il bottone "Pokécenter" è disabilitato quando `pokecenter_uses >= pokecenter_max`

## EXP e soldi

### EXP
Formula Gen V (semplificata):
```
exp_gained = floor((base_exp * enemy_level) / 7) * reward_multiplier
```
- `base_exp` da PokéAPI (`base_experience`)
- `reward_multiplier`: 1× selvatico, 2× allenatore, 4× capopalestra

Level up: quando `current_exp >= exp_to_next_level`.
Soglia livello: `exp_threshold(level) = (4 * level^3) / 5` (curva "medium fast").

### Soldi
```
money_gained = base_exp * enemy_level / 10 * reward_multiplier
```
Arrotondato al multiplo di 10 più vicino per leggibilità.

## Cattura

- Solo su Pokémon selvatici
- Disponibile dopo che il Pokémon nemico è sotto il 50% HP (semplificazione)
- Se team < 6: entra direttamente nel team
- Se team == 6: mostra UI di scelta sostituzione, il Pokémon scelto viene rimosso

---

## Evoluzione nella generazione nemici

1. Fetch `pokemon-species/{name}` → campo `evolution_chain.url`
2. Fetch `evolution-chain/{id}` → walk della chain
3. Per ogni nodo: se `min_level <= pokemon_level` → scala alla forma successiva
4. Continua fino a che non trova la forma corretta per il livello dato
5. Sia species che evolution_chain vengono salvate in `api_cache` IDB

## Struttura Rust (game-core)

```
game-core/src/
├── run/
│   ├── mod.rs          — RunState, RunPhase enum
│   ├── gym.rs          — GymProgress, sequenza allenatori
│   ├── rewards.rs      — calcolo EXP, soldi, level up
│   └── generator.rs    — generazione nemici (selvatici, allenatori, capo)
├── inventory.rs        — Inventory, ItemKind, use_item()
└── (esistenti: battle/, moves/, pokemon/, types/)
```

La logica di run vive in `game-core` (pura Rust, testabile).
La UI di navigazione vive in `frontend` (Leptos).
